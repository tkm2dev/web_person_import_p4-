# ระบบกรอกข้อมูลบุคคล ภ.4 — แยกจังหวัด + รวมส่วนกลาง

แยกไฟล์ให้ 12 จังหวัดใน ภ.4 กรอกข้อมูลของตัวเอง แล้วรวมกลับเข้าไฟล์ ภ.4 อัตโนมัติ
เพื่อให้ ภ.4 ส่งต่อส่วนกลางด้วย template เดิม

> 📖 **[docs/SYSTEM.md](docs/SYSTEM.md)** — เอกสารระบบฉบับสมบูรณ์ไฟล์เดียว
> สถาปัตยกรรม, คู่มือเจ้าหน้าที่, คู่มือผู้ดูแล, การขยายระบบ, ข้อจำกัด, สเปกภาพประกอบ

> ⚠️ **ห้าม commit ข้อมูลจริง** — `.gitignore` กัน `*.xlsx` และ `build/` ไว้แล้ว
> และในโหมด `LINK_EDIT` **spreadsheet id ของไฟล์จังหวัดเท่ากับรหัสผ่าน** จึงไม่อยู่ในซอร์สโค้ด

## สถาปัตยกรรม

```text
build/provinces/*.xlsx ──อัปโหลด+แปลง──▶  12 ไฟล์รายจังหวัด (Google Sheet)
                                                    │
                                              bindExisting()
                                                    ▼
                                         ไฟล์ ภ.4 ส่วนกลาง
                                    (ภ.4 เห็นทั้ง 12 จังหวัด)
                                                    ▲
                            syncAuto() ทุก 5 นาที ──┘
                                                    │
                                           exportForCentral()
                                                    ▼
                                       ภ.4_ส่งส่วนกลาง_*.xlsx
```

**ทำไมต้องแยกเป็นคนละ Spreadsheet** — Protected range ใน Google Sheets ป้องกันแค่การ *แก้*
ไม่ได้ป้องกันการ *อ่าน* คนที่เปิดไฟล์ได้ยังดึงข้อมูลทุกแท็บผ่าน API ได้อยู่ดี
การให้แต่ละจังหวัดเห็นเฉพาะข้อมูลตัวเองจึงต้องแยกเป็นคนละไฟล์เท่านั้น

## ขั้นตอนติดตั้ง

### 1. สร้างไฟล์ตั้งต้น

```powershell
python scripts/build_p4.py
```

ได้ `build/provinces/*.xlsx` (12 ไฟล์), `build/P4_SOURCE.xlsx`, `build/data_quality_report.md`
**อ่าน `data_quality_report.md` ก่อน** — มีรายการค่าที่สคริปต์แก้ให้ และช่องบังคับที่ยังว่างอยู่เยอะ

### 2. อัปโหลดไฟล์จังหวัดขึ้น Drive

สร้างโฟลเดอร์ใน Drive → อัปโหลด `build/provinces/*.xlsx` ทั้ง 12 ไฟล์
เปิดทีละไฟล์ → `ไฟล์ > บันทึกเป็น Google ชีต`

**ต้องแปลงเป็น Google Sheet** — สคริปต์อ่าน `.xlsx` ดิบไม่ได้ (ถ้าลืมแปลง สคริปต์จะฟ้องให้)

### 3. สร้าง Apps Script project

[script.google.com](https://script.google.com) → `โปรเจกต์ใหม่` → ลบโค้ดใน `Code.gs` ทิ้ง →
วางเนื้อหาจาก [apps_script/Code.gs](apps_script/Code.gs) ทั้งไฟล์ → บันทึก

ไฟล์เดียวจบ ไม่ต้องสร้างไฟล์อื่น ไม่ต้องแตะ `appsscript.json`

### 4. เลือกวิธีให้สิทธิ์

`fileId` ทั้ง 12 จังหวัดใส่ไว้ให้แล้ว เหลือเลือก `SHARE_MODE`

```js
SHARE_MODE: 'LINK_EDIT',   // ส่งลิงก์ให้จังหวัด แก้ไขได้เลย ไม่ต้องล็อกอิน ไม่ต้องเก็บอีเมล
// SHARE_MODE: 'EMAIL_ONLY',  // ไฟล์เป็น PRIVATE แชร์เฉพาะอีเมลใน editors
```

ไฟล์ ภ.4 ส่วนกลางเป็น PRIVATE เสมอไม่ว่าตั้งค่าเป็นอะไร เพราะรวมข้อมูลครบ 12 จังหวัด

### 5. รันทีละฟังก์ชัน

เลือกชื่อฟังก์ชันจาก dropdown ด้านบน → `เรียกใช้` → ครั้งแรกจะขอสิทธิ์ Drive + Sheets
(`ดูขั้นสูง > ไปที่ ... (ไม่ปลอดภัย)` เพราะเป็นสคริปต์ของตัวเอง)

| ลำดับ | ฟังก์ชัน | ทำอะไร |
|---|---|---|
| 1 | `auditSharing()` | รายงานสิทธิ์ปัจจุบันของทุกไฟล์ — อ่านอย่างเดียว |
| 2 | `verifyProvinceFiles()` | ตรวจว่าครบ 12 ไฟล์ มี 8 แท็บ หัวตารางตรง template — ไม่แก้อะไร |
| 3 | `bindExisting()` | สร้างไฟล์ ภ.4 + ติดตั้ง trigger + รวมข้อมูลรอบแรก |
| 4 | `hardenProvinceFiles()` | dropdown / ไฮไลต์ช่องบังคับ / ล็อกหัวตาราง + ตั้งสิทธิ์ตาม `SHARE_MODE` |
| 5 | `shareProvinceLinks()` | *(เฉพาะ LINK_EDIT)* ล็อกโฟลเดอร์ แล้วพิมพ์ลิงก์ 12 จังหวัดให้คัดลอกไปส่ง |

ข้อ 3 และ 4 ถ้าหยุดกลางคันเพราะเกิน 6 นาที **กด `เรียกใช้` ซ้ำได้เลย** — จำความคืบหน้าไว้

ข้อ 4 จำเป็นเพราะการ import `.xlsx` พา **data validation** มาด้วย
แต่ **ไม่พา conditional formatting และ protected range** มา

## โหมด LINK_EDIT — ส่งลิงก์ให้จังหวัดโดยไม่เก็บอีเมล

`shareProvinceLinks()` ทำ 3 อย่างตามลำดับ

1. **ล็อกโฟลเดอร์** เป็น "จำกัด" — ต้องทำก่อน ไม่งั้นคนที่ถือลิงก์โฟลเดอร์เห็นครบ 12 จังหวัด
2. ตั้งไฟล์จังหวัดแต่ละไฟล์เป็น "ทุกคนที่มีลิงก์ แก้ไขได้"
3. ตรวจว่าไฟล์ ภ.4 ยังเป็น PRIVATE ถ้าไม่ใช่จะตั้งกลับให้

แล้วพิมพ์ลิงก์ทั้ง 12 ออกมาใน Execution log พร้อมคัดลอกไปส่ง

### สิ่งที่ป้องกันไว้ให้

- หัวตารางล็อก แก้ไม่ได้
- ชีต `README` และ `Lists` ล็อกทั้งชีต (แหล่ง dropdown จะไม่ถูกแก้)
- `จังหวัด*` เลือกได้ค่าเดียว, `ประเภทข้อมูล*` ล็อกตามชีต
- ไฟล์ ภ.4 ไม่มีทางหลุดเป็นลิงก์สาธารณะ
- `snapshotMaster()` สำรองไฟล์ ภ.4 ทุกวันตี 1 เก็บ 14 ชุด (`SNAPSHOT_KEEP`)

### สิ่งที่ป้องกันไม่ได้ ต้องยอมรับ

- ประวัติการแก้ไขขึ้นว่า **"ผู้ใช้ที่ไม่ระบุชื่อ"** สืบไม่ได้ว่าใครแก้หรือลบ
- ใครถือลิงก์ **ลบแถวได้** และ `syncAuto()` เขียนทับไฟล์ ภ.4 ตามภายใน 5 นาที
  กู้ได้จากชุดสำรองรายวันเท่านั้น
- ใครถือลิงก์ **ดาวน์โหลด/คัดลอกทั้งไฟล์ได้**
- ลิงก์ที่หลุดออกไป (ส่งต่อ LINE, ประวัติเบราว์เซอร์, เครื่องที่ใช้ร่วมกัน) เพิกถอนทีละคนไม่ได้
  ต้องรัน `shareProvinceLinks()` ใหม่ทั้งชุดเพื่อออกลิงก์ใหม่

ถ้าจะเปลี่ยนใจภายหลัง ตั้ง `SHARE_MODE: 'EMAIL_ONLY'` ใส่อีเมลใน `editors`
แล้วรัน `lockDownProvinceFiles()` ตามด้วย `hardenProvinceFiles()` (รัน `resetHardenProgress()` ก่อน)

## การรวมข้อมูลทำงานอย่างไร

ไฟล์ ภ.4 มี **8 ชีตเดียวกัน คอลัมน์เดียวกัน** กับไฟล์จังหวัด ไม่มีคอลัมน์เกินแม้แต่คอลัมน์เดียว
`syncAuto()` แค่เอาแถวของทั้ง 12 จังหวัดมาต่อกันในชีตเดียวกัน แล้วใช้คอลัมน์ `จังหวัด*` แยกว่าแถวไหนของใคร

```text
ขอนแก่น/จิตเวชรักษา   4,699 แถว ─┐
ร้อยเอ็ด/จิตเวชรักษา   2,776 แถว ─┤
...                              ├─▶ ภ.4/จิตเวชรักษา  16,878 แถว (คอลัมน์เท่า template)
เลย/จิตเวชรักษา          197 แถว ─┘
```

รายละเอียดที่สำคัญ

1. **ตรวจว่ามีจังหวัดไหนแก้ไฟล์ไหม** ก่อน โดยอ่าน `lastUpdated` จาก Drive ถ้าไม่มีใครแก้เลยก็จบทันที
   ไม่ต้องเปิดอ่านทั้ง 12 ไฟล์
2. **แก้เฉพาะบล็อกของจังหวัดที่เปลี่ยน** ไม่ใช่ rebuild ทั้งชีต — เก็บ `TAB_COUNTS` ไว้ว่าแต่ละจังหวัด
   กินกี่แถวในแต่ละชีต แล้ว splice เฉพาะช่วงนั้น ถ้าจำนวนแถวที่จำไว้เพี้ยนจากชีตจริงเมื่อไร
   จะโยน error แล้ว **fallback ไป rebuild เต็มอัตโนมัติ** ไม่ยอมเขียนผิดที่
3. **จับคู่คอลัมน์ด้วยชื่อหัวคอลัมน์ ไม่ใช่ตำแหน่ง** — จังหวัดเผลอสลับคอลัมน์ก็ยังรวมถูกช่อง
4. **จังหวัดลบแถวไหน ฝั่ง ภ.4 หายตาม** ไม่มีข้อมูลผี — กู้ได้จาก `snapshotMaster()` เท่านั้น
5. **`จังหวัด*` และ `ประเภทข้อมูล*` เขียนทับด้วยค่าจากไฟล์ต้นทางและชื่อชีต** ไม่ใช่จากที่เจ้าหน้าที่พิมพ์
6. **แถวสำรองที่ pre-fill ไว้ไม่ถูกดึงมา** เพราะกฎ "แถวมีข้อมูล" ไม่นับ `ประเภทข้อมูล*` กับ `จังหวัด*`
7. `Dashboard` นับด้วย `COUNTIF` บนคอลัมน์ `จังหวัด*` ของแต่ละชีต, `_ทะเบียนไฟล์` เก็บลิงก์และเวลาซิงค์

### ทำไมต้อง incremental

บัญชี `@gmail.com` มีโควตา **trigger รวม 90 นาที/วัน** (Google Workspace ได้ 6 ชั่วโมง)

| | ต่อครั้ง | ต่อวัน |
|---|---:|---:|
| ไม่มีใครแก้ | ~4 วิ | 288 ครั้ง = 19 นาที |
| แก้ 1 จังหวัด — rebuild เต็ม (แบบเดิม) | ~106 วิ | 96 ครั้ง = **170 นาที** ❌ |
| แก้ 1 จังหวัด — splice บล็อก (แบบใหม่) | ~10 วิ | 96 ครั้ง = 16 นาที ✓ |

`รวมข้อมูลเดี๋ยวนี้` ในเมนูยัง force rebuild เต็มเสมอ ใช้เป็นปุ่ม reset ถ้าสงสัยว่าข้อมูลเพี้ยน

## แท็บในไฟล์ ภ.4

| แท็บ | คืออะไร |
|---|---|
| `README` | คำอธิบายจาก template |
| `ภาพรวม` | ทุกอย่างที่ต้องดูอยู่แผ่นเดียว — KPI + 2 ตาราง + 5 กราฟ |
| 8 แท็บข้อมูล | ข้อมูลทั้ง 12 จังหวัดต่อกัน คอลัมน์เท่า template |
| `Lists` | รายการ dropdown |
| `_ทะเบียนไฟล์` | ลิงก์ไฟล์จังหวัด + จำนวนแถว + เวลาแก้ไขล่าสุด |
| `_ข้อมูลกราฟ` *(ซ่อน)* | บล็อก 2 คอลัมน์ที่กราฟวงกลม/กราฟรายคอลัมน์ต้องใช้ |
| `_ประวัติรายวัน` *(ซ่อน)* | จำนวนแถวรายวัน ป้อนกราฟเส้น |

แท็บที่ไม่ใช่ template ทั้งหมดอยู่ใน `HELPER_TABS` — `exportForCentral()` ลบทิ้งก่อนส่งส่วนกลาง

### แท็บ ภาพรวม

```text
แถว 1-2   ชื่อเรื่อง + เวลาซิงค์ล่าสุด
แถว 4-5   KPI: รวมทุกประเภท | จังหวัดที่มีข้อมูล | ความครบถ้วน | ช่องบังคับที่ยังว่าง
แถว 7-21  ตารางจำนวนข้อมูล   จังหวัด × 8 ประเภท + รวม
แถว 23-37 ตารางความครบถ้วน   จังหวัด × 6 คอลัมน์ + เฉลี่ย (แถวรวมถ่วงน้ำหนัก)
แถว 40+   กราฟ 5 ตัว
```

| กราฟ | บอกอะไร |
|---|---|
| คอลัมน์ซ้อน | จำนวนข้อมูลรายจังหวัด แยกตามประเภท |
| วงกลม | สัดส่วนตามประเภทข้อมูล |
| แท่งแนวนอน | **% ความครบถ้วนของช่องบังคับ รายจังหวัด** |
| คอลัมน์ | **% ความครบถ้วนของช่องบังคับ รายคอลัมน์** |
| เส้น | จำนวนข้อมูลรวม รายวัน (ดูความคืบหน้า) |

กราฟความครบถ้วนคือตัวที่ใช้ตามงานจริง — รายงานคุณภาพชี้ว่า `วันที่บันทึก*` ว่าง 100%
ทั้ง `จิตเวชรักษา` และ `กลุ่ม5สี11กลุ่ม` และ `อำเภอ/เขต*` ว่าง 10,890 แถว

ทั้งหมดคำนวณด้วยสูตร `COUNTIF` / `COUNTIFS` ในชีต ไม่ใช่สคริปต์ จึงไม่กินโควตา trigger
ตำแหน่งคอลัมน์หาจากหัวตารางจริงทุกแท็บ — `เลขบัตรประชาชน` อยู่คอลัมน์ **K** ในแท็บที่ไม่มี
`ชื่อเล่น/ชื่ออื่น` แต่อยู่คอลัมน์ **L** ในแท็บอื่น ถ้า hardcode ไว้ ตัวเลข 2 แท็บจะผิดแบบเงียบๆ

## สิ่งที่เจ้าหน้าที่จังหวัดจะเห็น

- ข้อมูลเดิมของจังหวัดตัวเองเป็นแถวตั้งต้น พร้อมแก้ไข/กรอกเพิ่ม
- แถวสำรอง 500 แถวต่อชีต ที่ pre-fill `ประเภทข้อมูล*` + `จังหวัด*` ไว้แล้ว
- `จังหวัด*` ล็อกเป็นจังหวัดตัวเองอย่างเดียว, `ประเภทข้อมูล*` ล็อกตามชีต
- dropdown `คำนำหน้า` / `เพศ` / `ช่องทางรับเรื่อง` อ่านจากชีต `Lists`
- **ช่องบังคับ (`*`) ที่ว่างจะขึ้นพื้นสีแดง** เฉพาะแถวที่มีข้อมูลอื่นอยู่แล้ว
- หัวตารางถูกป้องกันไว้ แก้ไม่ได้

## ข้อควรรู้

**แถวไหนถูกนับว่ามีข้อมูล** — แถวที่มีค่าในคอลัมน์ใดก็ได้ *นอกเหนือจาก* `ประเภทข้อมูล*` และ `จังหวัด*`
กฎเดียวกันนี้ใช้ทั้งใน `build_p4.py` และ `Code.gs` แถวสำรองที่ pre-fill ไว้จึงไม่ถูกดึงไปรวม

**`resetAll()`** — ย้ายเฉพาะไฟล์ที่สคริปต์สร้างเอง (ไฟล์ ภ.4) ลงถังขยะ
ไฟล์จังหวัด 12 ไฟล์ที่อัปโหลดเองจะไม่ถูกแตะ ต้องตั้ง `CONFIG.ALLOW_RESET = true` ก่อน

**`resetHardenProgress()`** — ล้างความคืบหน้า เพื่อสั่ง `hardenProvinceFiles()` ทำใหม่ทั้งหมด

## ที่ต้องยืนยันกับส่วนกลาง

ดูรายละเอียดใน `build/data_quality_report.md` §7

- ชีต `Lists` เขียนว่า `บุคคลถวายฎีกา` แต่ README เขียน `บุคคลถวายฎีกาผ่านศูนย์ดำรงธรรม` — สคริปต์ใช้ค่าตาม `Lists`
- ชีต `เร่ร้อน` สะกดต่างจากภาพประเภทข้อมูลที่เขียน `เร่ร่อน` — คงตามต้นฉบับ ไม่แก้
- README ต้นฉบับอ้างถึงชีต `นำเข้ารวม` ที่ไม่มีอยู่จริง
- `Lists` ไม่มีรายการค่าของ `ระดับความเร่งด่วน` จึงยังเป็นช่องพิมพ์อิสระ (ไม่มี dropdown)
- `จิตเวชรักษา` ใช้หัวคอลัมน์ `ชื่อ` ไม่มี `*` ต่างจากชีตอื่นที่ใช้ `ชื่อ*` — คงตามต้นฉบับ
